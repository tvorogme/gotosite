Гранты

Привет! Знакомьтесь, это наши задания для грантовых конкурсов этой осенью. Код можно писать на произвольном языке, компилировать в Brainfuck-подобные нельзя. Не стесняйтесь задавать вопросы на school@goto.msk.ru c пометкой «Гранты. Осень 2017». Все задачи реальны, и их решение поможет реальным людям. Про них можно будет писать статьи, добавить в своё резюме, пользоваться ими (!) и так далее. Можно сделать значимую часть, поехать, рекрутировать на школе товарищей и доделать вместе и с ментором. Техническая документация приветствуется, и при должном уровне даже в отдельности от кода может принести вам дополнительные баллы.

Итоги мы подведем до 20 октября на нашем сайте. В конкурсе может принять участие любой желающий, за плагиат снимаем с конкурса, по решению проверяющих может быть запрошено онлайн-собеседование или серия уточняющих вопросов.

Блокчейн

GoToCoins

Сделать криптовалюту, майнинг в которой совершается не hashcash, а деланием чего-то полезного (например, написанием статей на хабр). С полезностью должно согласиться большинство, причём взвешенное (т.е. разные ноды имеют разный, априори заданный вес).

Криптовалюта должна быть основана на блокчейне (один сервер с одной базой данных обычно этому не удовлетворяет).

Мы планируем использовать такую валюту для выдачи поощрений и профита участникам. Кроме того, через смарт-контракты хочется закреплять обещания, проводить сертификацию, репутационные системы и голосования ("я точно закончу этот проект после школы/хакатона"). По итогам необходимо выслать решение до 18 октября включительно в виде ссылки на открытый github-репозиторий с исходным кодом и сопровождающей информацией.

Информационная безопасность

Криптографически умный замок

Часто СКУД-системы и прочие хипстерские NFC-замки уязвимы к простым в реализации атакам типа копирования карты.

Задача: сделать замок для хакспейса, которым сможет пользоваться бабушка, и который будет максимально безопасен и при этом дешёв. Чем более массово доступные компоненты используются - тем лучше.

Требования (сортировка по важности):
  • устойчивость к копированию ключа и replay-атакам
  • возможность открыть дверь удалённо
  • разные привилегии для разных пользователей
  • расширяемость до нескольких дверей (или понимание, как это возможно было бы сделать)
  • возможность дать временный доступ произвольному человеку
Результат – это софт + список компонентов с прайсами и ссылками для закупки. Делать непосредственно физический замок необязательно, достаточно софта, который смог бы дёргать условные функции open :: Door -> IO () и close :: Door -> IO (). В целях демонстрации можно просто писать в консоль. По итогам необходимо выслать решение до 18 октября включительно в виде ссылки на открытый github-репозиторий с исходным кодом и сопровождающей информацией.

Машинное обучение и анализ данных

Чат-бот GoTo для консультаций

Вам предлагается реализовать умного чат-бота для автоматизации ответов на частые вопросы по участию в школах, хакатонах и обучению. Конкретнее, бот должен сам находить в заданной базе FAQ вопрос, похожий по смыслу на вопрос пользователя и возвращать соответствующий ответ, или предложить ознакомиться с общими разделами FAQ.

Что нужно сделать:
  • Собрать синтетический набор данных (вопросы-дубликаты и ответы к ним). Для этого можно парсить форумы, вопросные сайты (toster.ru, stackoverflow.com), на которых повторяющиеся вопросы автоматически помечаются. Также допускается использование открытых данных (kaggle.com/quora/question-pairs-dataset) или ручная генерация собственными силами).
  • Спроектировать систему для хранения базы вопрос-ответ и ее пополнения в ходе общения с пользователями с модулем поиска семантически ближайшего вопроса с возможностью дообучения на новых вопросах.
  • Реализовать Telegram-бота для консультации по образовательным программам школы GoTo с возможностью расширения на другие темы. Чат-бот должен самостоятельно по реплике пользователя находить близкий по смыслу вопрос в базе знаний и возвращать соответствующий ответ, или сообщать, что такого вопроса нет и предлагать каталог тематик FAQ.
  • Для эффективного построения модуля поиска желательно предусмотреть функционал сбора обратной связи от пользователя после получения ответа на свой вопрос (например, предлагать оценивать релевантность ответа).
По итогам необходимо выслать решение до 18 октября включительно в виде ссылки на открытый github-репозиторий с исходным кодом и документацией, а также ссылка на запущенный рабочий чат-бот в Telegram в ней.
